## Agentic Data Studio

SqlKit 的 **Agentic Data Studio** 重新定义了数据库工作方式。无需从头编写 SQL，用自然语言描述需求 — AI 智能体自动读取 Schema、编写优化查询、可视化执行计划、实时修复错误。

- **自然语言转 SQL** — 基于实际表和字段名生成 Schema 感知的精准查询
- **SQL 优化** — 自动重写慢查询，以树形图可视化执行计划，成本高亮显示
- **错误修复** — AI 智能体自动诊断并修复 SQL 错误
- **安全机制** — 细粒度权限控制，危险操作需二次确认；凭据永不暴露给 LLM
- **自带密钥** — 支持 OpenAI、Anthropic、DeepSeek、Ollama 和 LM Studio。本地模型可完全离线使用

## 多数据库架构

SqlKit 通过五种适配策略支持 **50+ 种数据库**，覆盖从原生 Rust 驱动到灵活的 JDBC 桥接。没有其他开源 SQL 客户端能覆盖如此广度。

| 策略 | 支持的数据库 |
|----------|-----------|
| **原生**（Rust） | PostgreSQL、MySQL、SQL Server、SQLite、DuckDB、ClickHouse、Firebird、Oracle |
| **PG 协议兼容** | CockroachDB、Redshift、YugabyteDB、TimescaleDB、QuestDB、Vastbase 等 |
| **MySQL 协议兼容** | MariaDB、TiDB、OceanBase、StarRocks、Databend、ManticoreSearch 等 |
| **JDBC 桥接** | Snowflake、DB2、H2、BigQuery、Databricks、Hive、Hana、Teradata、Vertica、Exasol、Informix、Cassandra、TDengine 等 |
| **HTTP 桥接** | Trino、Presto |

## 跨引擎数据迁移

在任意支持的数据库之间传输数据，无需中间文件：

- **跨引擎迁移** — PostgreSQL 到 ClickHouse、Oracle 到 SQL Server、MySQL 到 BigQuery，自动类型映射
- **迁移向导** — 交互式界面，支持列自动映射
- **DDL 生成** — 为所选表生成 CREATE 语句，适配目标引擎 SQL 方言
- **SQL 文件执行** — 在目标数据库中执行 SQL 文件，支持可配置的错误处理策略
- **导入 / 导出** — CSV、JSON、JSONL 格式，支持百万级批量处理

## 常见问题

**SqlKit 支持哪些数据库？**
SqlKit 支持 50+ 种数据库，包括 PostgreSQL、MySQL、SQL Server、Oracle、SQLite、DuckDB、ClickHouse、Firebird、MariaDB、CockroachDB、Redshift、Snowflake、DB2、Trino、BigQuery、Databricks、Hive、Teradata 等。详见上方数据库兼容性表格。

**SqlKit 真的免费吗？**
是的。SqlKit 基于 Apache 2.0 开源协议，完全免费。所有功能包括 AI 智能体都不收费。您只需为使用的云端 LLM API 付费。

**SqlKit 可以离线使用吗？**
可以。桌面端的所有数据库操作均可完全离线运行。AI 功能需要网络连接（或通过 Ollama/LM Studio 使用本地模型）。

**SqlKit 会收集数据吗？**
不会。SqlKit 不收集任何遥测数据。自动更新功能仅检查 GitHub Releases 新版本 — 可在设置中关闭。您的凭据和查询记录始终保存在本地。

**SqlKit 与 DBeaver、DataGrip 有何不同？**
SqlKit 基于 Rust + Tauri 构建 — 无需 Java 运行环境、无 Electron 开销。AI 功能原生内置（非插件），支持 50+ 种数据库，单个小型安装包即可在 macOS、Windows、Linux 上运行。隐私优先，凭据加密本地存储。

**我可以使用自己的 AI 提供商吗？**
可以。SqlKit 支持 OpenAI、Anthropic、DeepSeek 以及任何 Ollama 兼容端点。在设置中配置提供商和密钥即可。AI 智能体不会将数据库凭据暴露给 LLM。

**SqlKit 支持 NoSQL 数据库吗？**
SqlKit 专注于 SQL 数据库。如果您需要管理 MongoDB、Elasticsearch、OpenSearch 或 DynamoDB，请查看我们的另一款开源产品 [**DocKit**](/zh/products/dockit/) — 专为 NoSQL 数据库打造的桌面客户端。
