## 常见问题

### DocKit 是否与 Dynobase 功能对等？

对于核心 DynamoDB 操作（查询、插入、更新、删除），是的。DocKit 提供：
- PartiQL 编辑器（语法高亮、自动补全）
- 可视化查询构建器
- 表浏览和数据编辑
- DynamoDB Local 支持

Dynobase 有更精美的 UI 和云同步，但 DocKit 覆盖了大多数开发者的需求。

### 我可以从 Dynobase 迁移到 DocKit 吗？

是的！虽然没有自动导入器，但您可以：
1. 在 DocKit 中重新创建连接配置（<5 分钟）
2. 将查询复制粘贴到 DocKit（如果已保存）
3. 两者都使用 AWS SDK，因此连接相同

### DocKit 是否发送遥测数据？

**否。** DocKit 不收集：
- 查询内容
- 数据库凭证
- 使用分析

所有遥测都是选择加入（崩溃报告等），默认禁用。

### 我可以同时使用 DocKit 和 Dynobase 吗？

是的！许多用户两者都安装，使用适合当前任务的工具。
